MCP Configuration Guide

Step-by-step setup guide to register and execute our custom high-fidelity KDM Boutique Showroom MCP server inside the official Stacklok ToolHive UI.
01

Navigate to "Add Custom Local Server"

Open your ToolHive desktop application. From the main dashboard panel, click on Add an MCP server and select Add custom local server from the dropdown list.

02

Select "Package Manager" Mode

In the config form, choose the Package manager containerization option. This lets ToolHive UI build a sandboxed virtual environment dynamically on top of the server package script.

03

Configure Local Environment Parameters

Populate the custom server parameters with the following precise targets:

Server Name KDM Showroom Local
Transport Protocol stdio
Package Protocol uvx
Package Target / Name /Users/nutkondee/VSCodeProjects/AIProjects/aiapps-guardrail-client/ToolHive/server.py
Standard Command-Line Arguments: To launch our server explicitly in native STDIO mode, enter --stdio inside the Command Arguments parameter box.
04

Install and Active

Click Install Server. ToolHive will resolve the package script, set up the proxy pipe, and launch the server. The card state on the dashboard will switch to ● Running.

01

Verify Python Script is Running Locally

Since our custom server is already running and listening on port 5001, you can register it instantly in the ToolHive UI without any container overhead!

# Check that uvicorn is running
curl http://localhost:5001/
02

Choose "Add Custom Remote Server"

In your ToolHive desktop app dashboard, click on the Add an MCP server dropdown and select Add custom remote server.

03

Enter the SSE Connection Endpoint

Register the server with the local SSE proxy bridge url:

• Server Name: KDM Showroom Remote
• Connection URL (SSE): http://localhost:5001/mcp
04

Active and Test Connections

Click Save Server. ToolHive UI will ping the SSE stream, establish the handshake, and automatically read all schemas for calculate_discount and get_stock_status tools!
